📐 Length-Extension Attack — How It Works
VULNERABLE SCHEME : TAG = SHA256( secret || message ) ATTACKER KNOWS : TAG, message, len(secret) [guessed via SA] PADDING APPENDED : 0x80 00...00 [length in bits as 64-bit LE] FORGED TAG : TAG' = SHA256_resume( TAG, extension ) ← no secret needed! RESULT : Forged contract passes server-side verification SECURE SCHEME : TAG = HMAC-SHA256( secret, message ) ← double hash blocks extension WHY SAFE : Outer H( k⊕opad || inner_hash ) cannot be resumed from known TAG
